오늘 대화 내용을 `.claude-history/` 폴더에 저장해주세요.

## 실행 단계

### 0단계: 환경 확인 (항상 실행)
- `.claude-history/` 폴더가 없으면 생성
- `.gitignore`에 `.claude-history/`가 없으면 추가

### 1단계: 기록 필요성 판단
현재 대화가 기록할 가치가 있는지 판단합니다.

**기록하지 않는 경우 (환경 확인 후 종료):**
- 단순 git 작업 요청 (커밋, push, PR 생성, 브랜치 전환 등)
- 단순 빌드/실행 명령 (gradle build, npm install 등)
- 파일 포맷팅, 린트 수정 등 단순 정리 작업
- 간단한 질문과 답변만 있는 경우
- 실질적인 코드 변경, 쿼리 작성, 트러블슈팅이 없는 경우
- `/save` 명령어 자체에 대한 설정/수정 작업

위 경우에 해당하면 "기록할 내용이 없습니다."라고 안내하고 종료합니다.

**기록하는 경우:**
- 새로운 기능 구현 또는 버그 수정
- 쿼리 작성 또는 데이터 분석
- 트러블슈팅 및 문제 해결 과정
- 아키텍처/설계 관련 논의
- 복잡한 로직 설명 또는 코드 리뷰
- 나중에 다시 참고할 가치가 있는 내용

### 2단계: 기존 파일 확인
- `.claude-history/` 폴더의 파일 목록을 읽어서 최근 파일들의 주제 파악
- 현재 대화의 핵심 키워드 추출 (예: 테이블명, 기능명, 이슈 키워드 등)

### 3단계: 주제 매칭 판단
**기존 주제와 연관된 경우 (업데이트):**
- 같은 테이블/기능/이슈에 대한 대화
- 같은 브랜치에서 작업 중
- 기존 파일에 언급된 키워드가 현재 대화에 50% 이상 등장

**새로운 주제인 경우 (새 파일 생성):**
- 완전히 다른 기능/테이블에 대한 대화
- 기존 파일과 키워드 겹침이 거의 없음

### 4단계: 파일 작성/업데이트
**업데이트 시:**
- 기존 파일의 관련 섹션에 새 내용 추가
- 중복 내용은 제외
- `## 업데이트 기록` 섹션에 날짜와 요약 추가

**새 파일 생성 시:**
- 파일명: `YYYY-MM-DD-주제.md`
- 주제는 대화에서 핵심 키워드 2-3개로 구성 (예: `zozotown-takerate-migration`)

## 파일 구조

```markdown
# 주제

**날짜**: YYYY-MM-DD
**브랜치**: (git에서 확인)
**관련 지라**: (브랜치명이나 대화에서 추출)

---

## 배경
(문제 상황, 목적)

## 핵심 내용
(주요 쿼리, 코드, 명령어 등)

## 트러블슈팅
(발견한 문제와 해결 방법)

## 관련 파일
(관련 코드 파일 경로)

## 참고사항
(기타 메모)

---

## 업데이트 기록
- YYYY-MM-DD: 최초 작성
- YYYY-MM-DD: XXX 내용 추가
```
